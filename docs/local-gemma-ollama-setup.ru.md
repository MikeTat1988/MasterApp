# Локальный `gemma4:e2b` через Ollama на Windows

Этот набор нужен для простого локального запуска `gemma4:e2b` на ноутбуке и для будущей интеграции с MasterApp без постоянного удержания модели в памяти.

## Где установлен Ollama

- Программа: `%LOCALAPPDATA%\Programs\Ollama`
- Основной exe: `%LOCALAPPDATA%\Programs\Ollama\ollama.exe`
- Локальные служебные файлы и логи: `%LOCALAPPDATA%\Ollama`

На этой машине подтвержден путь:

- `C:\Users\micha\AppData\Local\Programs\Ollama\ollama.exe`

## Где хранится модель

- Хранилище моделей по умолчанию: `%USERPROFILE%\.ollama\models`
- Manifest модели `gemma4:e2b`: `%USERPROFILE%\.ollama\models\manifests\registry.ollama.ai\library\gemma4\e2b`

На этой машине подтверждено:

- `C:\Users\micha\.ollama\models`

## Базовые команды Ollama

```bat
ollama pull gemma4:e2b
ollama run gemma4:e2b
ollama stop gemma4:e2b
```

## Как запустить и прогреть модель

Из корня репозитория MasterApp:

```bat
scripts\start_gemma.bat
```

Что делает скрипт:

- проверяет, что `ollama.exe` установлен
- проверяет, что модель `gemma4:e2b` уже загружена
- отправляет короткий warm-up запрос

После этого модель будет доступна быстрее для следующих запросов.

## Как остановить и выгрузить модель

```bat
scripts\stop_gemma.bat
```

Скрипт вызывает:

```bat
ollama stop gemma4:e2b
```

Замечание:

- после `stop` модель может на короткое время отображаться в `ollama ps` как `Stopping...`
- это нормально, через несколько секунд она обычно исчезает из списка

## Как протестировать через CLI

```bat
scripts\test_gemma.bat
```

Скрипт отправляет короткий русский prompt и печатает ответ модели.

## Как протестировать через API

Обычный запрос со стандартным поведением Ollama:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\test_gemma_api.ps1
```

Запрос с немедленной выгрузкой после ответа:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\test_gemma_api.ps1 -ImmediateUnload
```

Во втором случае в API передается:

```json
{
  "model": "gemma4:e2b",
  "prompt": "Коротко ответь по-русски: локальный API тест успешен.",
  "stream": false,
  "keep_alive": 0
}
```

Практический нюанс:

- `keep_alive=0` просит выгрузить модель сразу после ответа
- фактическая очистка памяти все равно может занять несколько секунд

URL локального API Ollama:

- `http://localhost:11434`

## Нужно ли держать модель постоянно загруженной

Нет. Это не рекомендуется для вашей задачи.

Лучший режим здесь такой:

- по умолчанию полагаться на стандартное idle-поведение Ollama
- прогревать модель вручную только когда нужно
- при необходимости выгружать сразу через `ollama stop gemma4:e2b`
- для одноразовых API-вызовов использовать `keep_alive=0`

## Как это использовать с MasterApp

Рекомендуемый паттерн для будущей интеграции:

- MasterApp отправляет обычные запросы к локальной модели через API Ollama
- локальная модель помогает с intent, структурированными ответами и простыми локальными задачами
- любые реальные действия на компьютере должны оставаться под approval-слоем MasterApp

То есть `gemma4:e2b` хорошо подходит как локальная, приватная и сравнительно дешевая модель для:

- быстрых локальных запросов
- черновых инструкций
- кратких резюме
- подготовки action proposals для компьютера

Но ее не стоит делать единственным агентом для сложного кодинга или для выполнения опасных действий без подтверждения.
